# 开发指南 & 规约手册 (Developer Guide)

## 📌 核心开发铁律（修改前必读！）

为防止在修改代码或更新功能时“修好这里、搞坏别的地方”，所有开发者（包括 AI 编码助手）在**每次进行代码修改前**，必须通读并严格遵守以下铁律：

### 1. 编码修改前的“三看”原则
* **一看 [docs/01-架构与设计.md](file:///d:/Code/Tools/LeiSu-Bypass/docs/01-架构与设计.md)**：重新确认你要修改的模块在整个数据流中的位置，防止打破现有的隔离防护和多级退化设计。
* **二看 CookieJar 隔离界限**：严禁在 `fetch_html_with_bypass` 中混合 live 域名与 odds 域名的 Cookie。对赔率走势（odds）的抓取修改，必须且只能使用 `GLOBAL_ODDS_CJ`；对主页及数据分析页抓取，必须且只能使用 `GLOBAL_CJ`。
* **三看 Fallback 退化机制**：凡是修改 H2H 历史交锋、近期战绩、伤停等抓取逻辑，**必须同时修改** JS 解密提取分支（即 `decrypt_shujufenxi_data` 成功后的分支）和 BeautifulSoup Fallback 兜底解析分支（即 `parse_h2h` / `parse_recent_results` ），确保两套解析方案在格式、Key 的拼写上完全保持对齐。

## 🛠️ 核心模块修改与防坏指引

### 1. 修改 WAF / HTTP 请求机制 (detail_scraper.py - fetch_html_with_bypass)
* **安全风险**：修改此函数极易导致雷速安全防护升级而大面积出现 403 / 405 / 503 封禁。
* **开发规约**：
  * 对 Cookie 的读写应加锁或在单线程/全局独占对象中进行。
  * 重试循环必须限制最大次数（当前为 3 次），且必须根据 HTML 长度判定（当前阈值为 `35000` 字节），防止陷入无限请求死循环或因加载不全导致报错。

### 2. 修改 JS 数据解密模块 (detail_scraper.py - decrypt_shujufenxi_data)
* **安全风险**：雷速可能不定期改变加密算法或 Caesar 偏置生成逻辑。
* **开发规约**：
  * 解密失败时**绝对不能 raise 抛出崩溃异常中断整个 Web 流程**。必须通过 `try...except` 捕捉，并在控制台/日志中打印 `print(f"Fallback to BeautifulSoup...")`，然后安全退回 BeautifulSoup 解析分支。

### 3. 修改比分质量审计与 Fallback 逻辑 (detail_scraper.py - get_complete_match_details)
* **安全风险**：如果把正常赛事的数据判定为了 bad data 强制 Fallback，会大幅降低解析性能（因为 BeautifulSoup 涉及复杂的 HTML DOM 解析）；如果漏掉了 bad data，则会再次出现全 `0:0` 战绩显示。
* **开发规约**：
  * **H2H 审计条件**：`matches` 数组必须长度大于 0，并且**全部**场次比分都为 `"0:0"`，才触发 Fallback。
  * **Recent 审计条件**：主客队近期战绩不为空，且**其中某一队全部战绩**的比分全为 `"0:0"`，即可触发 Fallback。
  * **比分补全处理**：对于当天正在进行的赛事，其比分有滞后性。修改 `current_score` 替换今天比分的后处理模块时，必须使用 `clean_team_name` 处理主客队名称，防止由于别名问题导致今天的实时比分未能合并。

### 4. 修改大模型提示词模块 (app.py - DEFAULT_SYSTEM_PROMPT / CRO_SYSTEM_PROMPT)
* **开发规约**：
  * 修改大模型 System Prompt 时，**必须保持双份同步**：一份在 `app.py` 的代码硬编码 `DEFAULT_SYSTEM_PROMPT` 中，另一份在 [ai_config.json](file:///d:/Code/Tools/LeiSu-Bypass/ai_config.json) 的 `system_prompt` 字段中。
  * 修改 `app.py` 中的提示词定义时，不要使用任何可能引起编辑器编码转换异常的特殊字符，读写时务必指定 `encoding='utf-8'`。

## 🧪 提质与编译验证规范

每次修改完任何 `.py` 文件后，必须在终端下执行以下两个步骤进行验证，严禁直接推送未验证代码：

1. **静态语法检查**：
   ```powershell
   python -m py_compile app.py detail_scraper.py
   ```
   输出为空代表编译通过。如果报错，必须当场修复。
2. **测试数据拉取检查**：
   可以使用本地测试脚本（如 `fetch_ai_report.py`）针对特定比赛拉取测试，确保不会因语法报错导致大模型收敛接口崩溃。
